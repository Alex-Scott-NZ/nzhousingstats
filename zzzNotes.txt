NZ Housing Statistics Project Summary
What We Built:
A complete NZ housing market data collection and dashboard system using TradeMe's public API.

Tech Stack:
Frontend: Next.js 15 with TypeScript, Tailwind CSS
Database: SQLite with Drizzle ORM (type-safe)
API: TradeMe Localities API (no authentication required)
Automation: Node-cron for scheduled data collection
Deployment: Ready for server deployment
Key Components Created:
1. Database Schema (lib/db/schema.ts)
weekly_snapshots - Raw API data with totals
location_snapshots - Flattened region/district/suburb data
Stores complete NZ geographic hierarchy with property counts
2. Data Collection (lib/data-collection.ts)
collectPropertyData() - Fetches from TradeMe API, stores in database
getTotalsByLocationType() - Dashboard summary stats
getLocationsWithFilters() - Queryable location data
getLatestSnapshots() - Recent collection history
3. TypeScript Types (lib/types/trademe.ts)
Complete type definitions for TradeMe API responses
Database schema types
Query parameter types
4. Next.js Dashboard (src/app/page.tsx + PropertyDashboard.tsx)
Summary cards (total listings, regions, districts, suburbs)
Top regions table with rankings and percentages
Recent data collection history
Last updated timestamps
5. Automated Data Collection
15-minute cron job for continuous data collection
Configurable for different listing types (buy/rent/commercial)
Production-ready with PM2 support
Current Data:
34,743 NZ property listings collected
2,397 location records (every region, district, suburb in NZ)
Real-time updates every 15 minutes
Complete geographic coverage of New Zealand
